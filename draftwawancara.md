# Catatan Wawancara - Idin Iskandar, S.Kom

### Pembukaan
"Sebelumnya saya mohon maaf untuk Bapak/Ibu. Semalam saya telah menyiapkan catatan untuk wawancara ini agar penyampaian saya lebih terarah. Karena saya terkadang merasa grogi saat berbicara langsung, mohon izin untuk membacakan draft yang telah saya susun rapi ini."

---

### 1. Filosofi Pengembangan (Modern Development)
"Sebagai developer di era modern, fokus utama saya saat ini adalah pada perancangan arsitektur sistem, efisiensi database, dan keamanan aplikasi. Saya berperan sebagai arsitek sistem yang menyusun logika dan alur kerjanya.

Untuk penulisan baris kode atau sintaks dasarnya, saya tidak mengetik manual dari nol, melainkan berkolaborasi dengan AI Assistant khusus."


"Sebagai gambaran, ini adalah **IdinCode**, asisten AI yang saya kustomisasi sendiri untuk alur kerja saya. Bapak/Ibu bisa lihat, alat ini dirancang khusus untuk arsitektur sistem hingga *frontend/backend*. Dengan alat ini, kalau perusahaan butuh sistem operasional atau pendataan konsumen (seperti data *fleet*), saya bisa membangunnya jauh lebih cepat dari developer tradisional tanpa mengurangi kualitas sistemnya."

### 2. Keamanan & Efisiensi Infrastruktur
"Saya sangat menekankan keamanan data dan efisiensi biaya operasional.
* **Contoh Implementasi:** Pada projek yang sedang saya pegang, saya merancang sistem agar aset sensitif seperti API Key tidak disimpan di dalam aplikasi HP yang rawan dibongkar (*reverse engineering*).
* **Solusi Teknis:** Saya membangun *Proxy Serverless* sendiri di cloud agar aplikasi aman dari spammer, namun dengan biaya operasional yang tetap nol rupiah."

### 3. Transparansi & Gaya Kerja
"Saya jujur bahwa dalam situasi wawancara tatap muka seperti ini, saya terkadang merasa cukup gugup. Itulah mengapa selama ini saya lebih nyaman mengambil projek secara *freelance* dari rumah. Di lingkungan *freelance*, komunikasi kami lebih banyak berbasis teks, panggilan telepon, atau video call yang berfokus langsung pada *progress* produk. 

Namun, saya sangat terbuka untuk berdiskusi langsung jika tujuannya adalah membangun fondasi digital perusahaan dari nol."

### 4. Visi untuk Perusahaan
"Karena saya terbiasa membangun projek secara mandiri, saya bisa menangani alur kerja dari hulu ke hilir—mulai dari mendesain antarmuka (UI/UX) yang modern, mengintegrasikan sistem, hingga memastikan keamanannya siap rilis. Jika di Yamaha Arista Johar ini tim developer-nya belum ada, saya siap menjadi pionir untuk merancang standardisasi sistem IT-nya terlebih dahulu."
